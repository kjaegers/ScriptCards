Changelog for AuraTriggers API Mod

Potential ToDos:
- Add color override such as "On:Auraname" or "Off:AuraName" to allow the aura to not be tied to a Roll20 Aura value.
- Add radius overrides
- Aura library handout? Instead of GMNotes field?

V 0.8:
- Properly handles an icon not being specified. Previously an aura without and icon was being ignored.
- Added support for walls blocking auras using Path Math and "pathv2" lighting objects. Legacy "path" objects are not supported.
- Added "ignoreWalls" option to aura definitions to allow certain auras to bypass wall blocking even if Path Math is enabled.
- Minor code cleanup and bug fixes.

V 0.7:
- Added comprehensive error handling throughout the script to prevent crashes from undefined or null object access.
- Enhanced error logging throughout for better debugging without breaking script execution.
- Added validation for findObjs results before accessing array elements.
- Added chatActonOnStartTurn and chatActionOnEndTurn triggers for effects that trigger based on these events.
- Added corresponding VFX and sound effect support for start turn/end turn events as well as while inside events.

V 0.6:
- Add VFX to applied/removed auras, potentiallly on the aura token and on the affected token.
- Add sound playback to applied/removed auras.
- Registers with both ScriptCards and TokenMod for observer support.

V 0.5
- Added `!at-report` command to log cached active auras, including aura name and attribute filter values.
- Added `attributeFilter` support in aura JSON to gate aura application based on character attributes.
- Added `NewGetAttrByName` helper (using `findObjs`) to avoid `Error: No attribute or sheet field found for character_id` log spam.
- Fixed runtime errors when filtered attributes are missing by safely normalizing undefined/null attribute values.

V 0.4
- Added cleanup for destroyed aura-source tokens so impacted tokens can process exit actions and status-marker removal.
- Fixed aura tracking to correctly handle multiple same-color aura definitions by including JSON record index in aura_id.
- Added cleanup for disabled auras in change:graphic handler; when aura configuration is modified, any auras that are disabled will properly trigger exit actions and marker removal without re-triggering enter actions for unchanged auras.

V 0.3 
- Added layer-based targeting via the aura `toLayers` setting.
- Added support for multiple aura definitions sharing the same color; all matching entries are now loaded and processed.
- Updated chat action variable replacement so `[ANAME]` resolves to the aura effect name and `[ATNAME]` resolves to the aura source token name.
- Added internal active-aura source detection helper logic used when rechecking page-wide overlaps.
- Removed unused placeholder config settings from the script.

V 0.2 
- Moving tokens with auras now triggers an aura check for all tokens on the same page.
- Removed lots of debugging messages
- Added a manual !at-checkall command to trigger an aura check for all tokens on all pages.
- Added a call to build all aura lists on API ready.
- Added applySelf option to allow tokens to trigger their own auras.
- Now handle add:graphic events to ensure new tokens are checked for aura overlap.

V 0.1 
- Initial development and testing. Not ready for use in live games yet.
